# 모델 비교 전략

## 목적

현재 기본 검색 경로와 향후 후보 모델을 같은 사진 라이브러리에서 비교할 수 있도록 기준을 정리한다.

## 현재 기본 구성

- DB 기본값: `data/multilingual.db`
- 이미지 임베딩: `openai/clip-vit-base-patch32`
- 한국어 직접 임베딩: `sentence-transformers/clip-ViT-B-32-multilingual-v1`
- 번역 fallback: 사용 안 함

## 향후 비교 대상 예시

- 더 큰 다국어 이미지-텍스트 모델
- 이미지 임베딩까지 교체한 완전 다국어 스택
- 재정렬 단계를 포함한 2단계 검색 구성

## 비교 절차

1. 현재 기본 구성을 기준선으로 둔다.
2. 후보 모델별로 별도 DB를 만든다.
3. 동일한 질의 세트를 각각 검색한다.
4. 상위 5장 또는 10장의 체감 정확도를 비교한다.

## 권장 질의 세트

- `노란색이 많이 들어간 풍경`
- `겨울 호수와 나무가 있는 장면`
- `도시 야경`
- `꽃이 많은 정원`
- `음식이 놓인 테이블`
- `a landscape with a lot of yellow`

## 평가 기준

- 한국어 질의 상위 결과의 관련성
- 영어 질의 성능 유지 여부
- 상위 결과의 일관성
- 초기 검색 지연시간
- 인덱싱 속도와 운영 복잡도

## 운영 권장

- 비교 실험용 DB는 기본 DB와 분리한다.
- 새로운 이미지 모델을 도입할 때는 별도 DB를 만든다.
- 비교가 끝나기 전까지는 현재 기본 DB를 덮어쓰지 않는다.
