# 결정 0008: 실행 경고는 줄이고, 실제 오류는 명확히 남긴다

## 상태

채택

## 배경

모델 로딩 중 출력되는 경고가 많으면 사용자는 어떤 메시지가 중요한지 구분하기 어렵다. 특히 BLIP의 tie-word-embeddings 경고처럼 기능을 막지 않는 메시지는 로그를 불필요하게 시끄럽게 만든다.

## 결정

다음 원칙을 적용한다.

- BLIP tie-word-embeddings 경고는 설정 보정과 로그 필터로 줄인다.
- 실제 실행 실패로 이어지는 오류는 숨기지 않는다.
- 자주 보게 되는 경고와 오류의 의미는 문서로 남긴다.

## 결과

### 장점

- 사용자가 중요한 오류를 더 쉽게 식별할 수 있다.
- 정상 동작 중 불필요한 경고 노이즈를 줄일 수 있다.
- 운영 중 문제 대응 기준이 명확해진다.

### 단점

- 특정 경고를 필터링하는 로직을 유지해야 한다.
- 라이브러리 메시지 형식이 바뀌면 필터도 조정이 필요할 수 있다.

## 후속 작업

- 자주 발생하는 다른 경고도 분류해서 문서에 추가
- 필요하면 verbose 로그 모드 검토
