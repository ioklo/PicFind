# 결정 0001: Python CLI 기반 MVP

## 상태

채택

## 배경

저장소가 비어 있는 상태에서 시작했고, 첫 결과물은 로컬 PC에서 빠르게 실행 가능해야 하며 내부 구조도 단순해야 한다.

## 결정

첫 버전은 다음 조합의 Python CLI 애플리케이션으로 만든다.

- `transformers` 로 BLIP 캡션 생성과 CLIP 임베딩 수행
- `SQLite` 단일 저장소 사용
- 정규화된 임베딩을 데이터베이스에 직접 저장
- 초기 검색은 brute-force 코사인 유사도 방식 사용
- 인덱싱 중에는 배치 단위로 중간 저장 수행

## 결과

### 장점

- 인프라가 단순하다.
- 로컬 설치와 실행이 쉽다.
- 백업과 점검이 쉽다.
- 외부 서비스 없이 동작한다.
- 긴 인덱싱 작업 중간에 중단되더라도 저장 손실을 줄일 수 있다.

### 단점

- 데이터가 많아질수록 검색 시간이 늘어난다.
- 임베딩을 DB에 저장하므로 용량이 커진다.
- 기본 모델 기준으로 다국어 품질이 제한된다.

## 후속 작업

사진 수가 커지면 FAISS 같은 전용 벡터 인덱스를 도입하고, 한국어 검색을 위해 다국어 검색 모델도 검토한다.
