# 결정 0011: caption 벡터화는 후속 단계로 도입 검토

## 상태

제안됨

## 배경

현재 검색은 이미지 임베딩만 사용한다. caption은 DB의 문자열 필드로만 저장되며, 검색 점수 계산에는 반영되지 않는다.

Florence-2로 더 자세한 caption을 생성하게 되면서, caption 자체를 텍스트 임베딩으로 변환해 검색 보조 신호로 활용할 가치가 생겼다.

## 제안

다음 구조를 후속 단계로 검토한다.

- 이미지 임베딩은 기존대로 유지한다.
- caption 생성 시 caption 텍스트 임베딩도 함께 저장한다.
- 검색 시 이미지 점수와 caption 점수를 함께 사용한다.

예시 전략:

- `final_score = 0.7 * image_score + 0.3 * caption_score`
- 또는 이미지 검색 top-k 추출 후 caption 점수로 재정렬

## 기대 효과

- 문장 질의와 caption 텍스트가 직접 비교되므로 언어적 단서가 더 잘 반영될 수 있다.
- 장면 설명, 객체 관계, 분위기 같은 표현이 caption에 잘 담긴 경우 검색 품질이 좋아질 수 있다.
- 이미지 임베딩 단독 검색의 약점을 보완할 수 있다.

## 우려 사항

- caption 품질이 낮으면 오히려 노이즈가 될 수 있다.
- 작은 디테일이나 잘못 생성된 설명이 검색을 왜곡할 수 있다.
- 추가 임베딩 저장과 점수 결합 로직이 필요하다.

## 현재 결론

caption vectorization은 유망하지만, 오늘 시점에서는 구현하지 않는다.

현재 기본 전략은 다음과 같다.

- 검색 핵심: 이미지 임베딩
- caption 역할: 저장 및 UI 표시용 보조 정보
- 후속 단계: caption 임베딩 저장 및 결합 점수 실험

## 후속 작업

- caption 임베딩 저장용 스키마 추가 검토
- image/caption 점수 결합 방식 비교
- 한국어 질의에서의 실효성 평가
