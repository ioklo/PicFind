# 결정 0009: baseline preset은 제거하고 multilingual 구성을 기본값으로 유지

## 상태

채택

## 배경

baseline 과 multilingual 을 병행 비교한 결과, 현재 사용 시나리오에서는 multilingual 구성이 더 실용적이었다. preset 구조는 비교에는 도움이 되었지만, 일상 사용 관점에서는 설정 복잡도를 높였다.

## 결정

다음 원칙을 적용한다.

- `baseline` preset은 제거한다.
- 기본 DB는 `data/multilingual.db` 로 유지한다.
- 한국어 직접 임베딩이 기본 경로다.
- 별도 비교 실험이 필요하면 DB 경로와 모델 옵션을 직접 지정한다.

## 결과

### 장점

- 설정이 단순해진다.
- 사용자가 어떤 구성을 선택해야 하는지 고민할 필요가 줄어든다.
- 실제 사용 흐름이 한국어 검색 중심으로 정리된다.

### 단점

- baseline 과의 즉시 전환 비교 기능은 사라진다.
- 비교 실험 시 사용자가 DB 경로를 직접 관리해야 한다.

## 후속 작업

- compare 전용 도구가 필요하면 별도 명령으로 재도입 검토
- 이미지 모델 교체 실험 절차는 문서로 유지
