# 결정 0007: 한국어 질의는 다국어 CLIP 직접 임베딩을 기본으로 사용

## 상태

채택

## 배경

기본 CLIP 모델은 영어 중심이라 한국어 질의 정확도가 일관되지 않았다. 초기에는 번역 fallback도 함께 두었지만, 구조가 복잡해지고 동작 예측이 어려워졌다.

## 결정

다음 전략을 기본값으로 채택한다.

- 한국어 문자가 포함된 질의는 `sentence-transformers/clip-ViT-B-32-multilingual-v1` 로 직접 임베딩한다.
- 이 모델은 기본 이미지 임베딩 모델인 `openai/clip-vit-base-patch32` 와 맞는 텍스트 임베딩 공간을 사용한다.
- 번역 fallback 경로는 제거한다.
- CLI와 웹 UI에서 실제 사용한 검색 경로를 보여준다.

## 결과

### 장점

- 구조가 단순해진다.
- 번역 품질에 의존하지 않고 한국어를 직접 처리할 수 있다.
- 현재 기본 이미지 인덱스를 유지하면서 한국어 검색 품질을 개선할 수 있다.

### 단점

- 첫 한국어 검색 시 다국어 텍스트 모델 다운로드가 필요하다.
- 기본 CLIP 이미지 임베딩과의 호환성 전제를 유지해야 한다.
- 다국어 모델 품질이 모든 한국어 표현을 완벽히 해결하지는 못한다.

## 후속 작업

- 다국어 이미지-텍스트 검색 모델 추가 평가
- 재정렬 전략 검토
