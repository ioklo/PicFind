# 결정 0004: 검색용 웹 UI는 Streamlit으로 제공

## 상태

채택

## 배경

문장 검색 결과를 경로만으로 확인하는 것은 불편하다. 사진 검색 도구는 상위 결과를 빠르게 시각적으로 검토할 수 있어야 한다.

## 결정

첫 웹 UI는 Streamlit으로 구현한다.

- 검색 질의를 입력할 수 있어야 한다.
- 상위 결과의 이미지 썸네일을 보여줘야 한다.
- 점수, 파일 경로, 캡션을 함께 표시한다.
- 인덱싱은 계속 CLI에서 수행하고 UI는 조회 중심으로 유지한다.

## 결과

### 장점

- 구현이 빠르고 유지보수가 단순하다.
- 로컬에서 바로 실행 가능하다.
- MVP 단계에서 검색 품질 검토에 충분하다.

### 단점

- 고급 이미지 브라우징 기능은 제한적이다.
- 대규모 라이브러리 탐색 UI로는 확장이 필요하다.

## 후속 작업

- 폴더/날짜 필터 추가
- 썸네일 캐시 추가
- 결과 상세 보기와 유사 이미지 탐색 추가
