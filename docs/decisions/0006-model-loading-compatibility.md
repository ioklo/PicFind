# 결정 0006: 모델 로딩은 safetensors 우선, torch 2.6 이상 권장

## 상태

채택

## 배경

최근 `transformers` 는 `torch.load` 관련 보안 취약점 대응으로 일부 체크포인트 로딩에 더 엄격한 버전 조건을 요구한다. `torch 2.5.x` 이하 환경에서는 공개 모델 다운로드가 끝나도 로딩 단계에서 실패할 수 있다. 또한 `transformers` 버전에 따라 CLIP 임베딩 호출 결과가 순수 텐서가 아니라 모델 출력 객체로 반환될 수 있다.

## 결정

다음 원칙을 적용한다.

- 모델 로딩 시 `safetensors` 형식을 우선 시도한다.
- 현재 환경이 낮은 `torch` 버전 때문에 차단되면, `torch>=2.6` 업그레이드 안내가 포함된 오류를 낸다.
- 프로젝트 의존성 최소 버전도 `torch>=2.6` 으로 올린다.
- 임베딩 정규화 로직은 텐서와 모델 출력 객체를 모두 처리한다.

## 결과

### 장점

- 가능한 경우 더 안전한 포맷으로 모델을 읽는다.
- 사용자가 원인을 빠르게 이해하고 복구할 수 있다.
- 최근 `transformers` 동작과 맞는 기준을 프로젝트에 명시할 수 있다.
- CLIP 출력 타입 차이로 인한 런타임 오류를 줄일 수 있다.

### 단점

- 기존 `torch 2.5.x` 환경은 업그레이드가 필요하다.
- 일부 모델은 safetensors가 없어서 결국 torch 업그레이드가 필요할 수 있다.

## 후속 작업

- 인덱싱 시작 시 torch 버전 체크 추가 검토
- 캡션 모델 실패 시 자동 강등 정책 검토
